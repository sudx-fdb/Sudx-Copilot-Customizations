First read the documentation rules in [documentation.instructions.md](../instructions/documentation.instructions.md).

Use the skill `documentation-full` to create or update the complete project documentation for GitHub.

This includes:
- README.md (project description, features, installation, usage, structure)
- Code docs (`docs/code_docs/`) -- technical function descriptions
- Usage docs (`docs/usage_docs/`) -- user guides
- Both `inhalt.md` files
- Additional GitHub docs (Contributing, Changelog) if present
