Analyze the project structure and provide a comprehensive overview:

1. **Folder Structure**: What is where? Which conventions are used?
2. **Technology Stack**: Languages, frameworks, libraries, tools
3. **Architecture Patterns**: MVC, microservices, monolith, etc.?
4. **Entry Points**: Where does the application start? Where does data flow in?
5. **Module Dependencies**: Which modules depend on each other?
6. **Configuration**: Which config files exist? What is configured?
7. **Specifics**: Unusual patterns, custom tooling, special conventions

Present the result clearly with a folder structure tree and explanations.
